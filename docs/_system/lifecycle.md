
# Lifecycle Rules

## Phase lifecycle

planned → active → complete

## Sprint lifecycle

planned → active → complete

A sprint is complete when all items are either:
- complete, or
- explicitly deferred to another sprint (recorded in the sprint doc)

## Feature lifecycle

planned → in_progress → review → complete
and can enter blocked before completion.

## Bug lifecycle

Use these statuses:
- planned (open)
- in_progress (fixing)
- complete (resolved)

## Refactor lifecycle

planned → in_progress → complete

## ADR lifecycle

Use these statuses:
- planned (proposed)
- complete (accepted)
- deprecated (superseded)
